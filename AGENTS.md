## 📜 Development Rules (Antigravity)

1. **Size Limit**: **Max 150 lines per chapter file** (where possible). Split long chapters into sub-sections.
2. **Clean Assets**: All images must be compressed and placed in `docs/public/`.
3. **Strict Metadata**: Every chapter must have a YAML frontmatter for search optimization.
4. **Validation**: Run `docs:build` before every commit to ensure zero broken links.

## 🤝 Interaction Protocol
- Follow the **Plan -> Act -> Validate** cycle for new chapters.
- The Agent must check narrative consistency before committing new prose.
